# Развёртывание LexChat на бесплатных хостингах

Это руководство поможет вам запустить LexChat полностью бесплатно, используя:
- **InfinityFree** – для PHP, MySQL, файлов
- **Render** – для WebSocket-сервера (Node.js)
- **Gmail SMTP** – для отправки кодов подтверждения
- **UptimeRobot** – чтобы WebSocket-сервер не засыпал

---

## 1. Регистрация и подготовка InfinityFree

1. Зарегистрируйтесь на [InfinityFree](https://www.infinityfree.com/)
2. После входа перейдите в **Control Panel** → **Account Manager** → **My Websites**
3. Нажмите **Create Account** → выберите бесплатный домен (например, `ваш-сайт.rf.gd`)
4. Запомните или запишите **FTP-логин** и **пароль** (они понадобятся для файлового менеджера)
5. Перейдите в **Control Panel** → **MySQL Databases** → **Create New Database**
6. Скопируйте **Database Name**, **Username**, **Password** (сохраните их)

---

## 2. Загрузка PHP-файлов через браузер (без FTP)

InfinityFree предоставляет веб-файловый менеджер. Вот как загрузить файлы прямо из браузера:

### Шаг 2.1. Вход в файловый менеджер

- В панели управления InfinityFree найдите раздел **File Manager** (или перейдите по прямой ссылке: `https://files.infinityfree.com`)
- Войдите, используя **FTP-логин** и **пароль** (те же, что вы получили при создании сайта)

### Шаг 2.2. Загрузка архива с проектом

**Способ A – через ZIP-архив (рекомендуется для большого количества файлов):**

1. На локальном компьютере упакуйте все файлы проекта в ZIP-архив (НЕ включайте папку `uploads` и другие временные файлы, если они есть)
2. В файловом менеджере откройте папку `htdocs` (это корень вашего сайта)
3. Нажмите кнопку **Upload** → выберите ZIP-файл
4. После загрузки нажмите на ZIP-файл правой кнопкой → **Extract** (извлечь)
5. Дождитесь распаковки, затем удалите ZIP-файл

**Способ B – по одному файлу (для мелких правок):**

- Нажмите **New File** или **Upload** и выберите файлы по очереди
- Не забудьте загрузить все PHP, JS, JSON, PNG, SQL и другие необходимые файлы

### Шаг 2.3. Настройка прав доступа

В файловом менеджере найдите следующие папки и установите для них права **777** или **755** (чтобы PHP мог записывать файлы):

- `uploads/` (и все вложенные: `avatars/`, `group_avatars/`, `photo/`, `files/`, `records/`)
- сами файлы `colors.json`, `admin_settings.json` – они должны быть доступны для записи (обычно 666 или 755)

Как изменить права: выделите папку/файл → **Permissions** → укажите **777** для папок и **666** для JSON-файлов.

## 3. Импорт структуры базы данных

### Через phpMyAdmin (бесплатно предоставляется InfinityFree)

1. В панели управления InfinityFree найдите **MySQL Databases** → напротив вашей БД нажмите **phpMyAdmin**
2. Войдите с данными базы (имя, пользователь, пароль)
3. Нажмите на название вашей БД слева
4. Перейдите в **Import** (Импорт)
5. Нажмите **Choose File** и выберите файл `bd.sql` из вашего проекта
6. Нажмите **Go** – таблицы будут созданы автоматически

## Альтернатива: выполнить SQL-запрос вручную

Если импорт не работает, скопируйте содержимое `bd.sql` и вставьте его в **SQL** вкладке phpMyAdmin.

##4. Настройка подключения к БД  
##
Отредактируйте файл `connect.php` (через файловый менеджер или FTP):

```php
$host = 'sql312.infinityfree.com';           // обычно sql.ваш-хост.com, уточните в панели
$dbname = 'if0_xxxxxxxx_baza';               // имя вашей БД
$user = 'if0_xxxxxxxx';                      // пользователь БД
$pass = 'ваш_пароль_базы';                   // пароль БД


Если вы используете переменные окружения, оставьте как есть и укажите их в панели InfinityFree
(не все бесплатные хостинги поддерживают .env).

## 5. Настройка почты (SMTP)
##

Отредактируйте файл api.php, найдите блок отправки кода:

$mail->Host       = 'smtp.gmail.com';
$mail->Username   = 'ваша_почта@gmail.com';
$mail->Password   = 'пароль_приложения_gmail';

Важно: Для Gmail нужно создать пароль приложения (если включена двухфакторная аутентификация)
или использовать обычный пароль (менее безопасно).
Инструкция: Как создать пароль приложения Gmail

Вы также можете использовать другой SMTP-сервер (например, Яндекс, Mail.ru) – измените хост и порт.

## 6. Развёртывание WebSocket-сервера на Render
##

Зарегистрируйтесь на Render (через GitHub)

Нажмите New + → Web Service

Подключите ваш репозиторий GitHub с проектом (или загрузите папку server отдельно)

Настройте:

Name: lexchat-websocket

Environment: Node

Build Command: npm install

Start Command: node server.js

Free Plan – выберите бесплатный тариф

Нажмите Create Web Service

После деплоя скопируйте URL сервера, например:
https://lexchat-websocket.onrender.com

Обновление настроек в админке
Зайдите в ваш чат: https://ваш-сайт.rf.gd/admin.php

Авторизуйтесь (пароль: admin123 – смените позже)

Перейдите на вкладку Нагрузка

В поле 🌐 WebSocket сервер вставьте URL вашего Render-сервера

Нажмите Сохранить настройки нагрузки

Теперь клиенты будут подключаться к вашему WebSocket-серверу.

## 7. Настройка UptimeRobot (чтобы Render не засыпал)
##

Render бесплатный сервер засыпает через 15 минут без активности. Чтобы этого избежать:

Зарегистрируйтесь на UptimeRobot (бесплатно – до 50 мониторов)

Нажмите Add New Monitor

Monitor Type: HTTP(S)

Friendly Name: LexChat WebSocket

URL: https://lexchat-websocket.onrender.com/healthz

Monitoring Interval: 5 minutes (или 10 minutes)

Нажмите Create Monitor

Дополнительно можно добавить монитор на PHP-сайт:
https://ваш-сайт.rf.gd/keepalive.php (если вы создали такой файл) с интервалом 15 минут.

## 9. Смена пароля администратора
##

Откройте admin.php в редакторе. Найдите строку:

php
$admin_password_hash = '240be518fabd2724ddb6f04eeb1da5967448d7e831c08c8fa822809f74c720a9';
Сгенерируйте новый хеш SHA256 для вашего пароля (например, на онлайн-генераторе) и замените.

Или используйте встроенную функцию:

php
password_hash('ваш_пароль', PASSWORD_DEFAULT);
Но учтите, что в текущей версии админка использует SHA256 (легче вписать).
Для усиления безопасности лучше переписать на password_verify().

## 10. Финальные штрихи
##

Измените иконку icon.png и фон fonDefault.png на свои

Настройте цвета и фон через админ-панель (вкладка Оформление)

Установите разумные интервалы polling на вкладке Нагрузка (60 сек для чатов, 120 сек для рассылок)

Проверьте работу в двух браузерах, отправьте файл, голосовое сообщение

Устранение неполадок
Ошибка подключения к БД
Проверьте параметры в connect.php

Убедитесь, что БД создана, а пользователь имеет права

WebSocket не подключается
Проверьте URL в админ-панели (вкладка "Нагрузка")

Откройте консоль браузера (F12) – там будут сообщения об ошибках

Проверьте, что сервер на Render запущен (логи в панели Render)

Файлы не загружаются
Проверьте права на папку uploads/ (должна быть 777)

Убедитесь, что лимит на загрузку файлов (5 МБ) не превышен

Не приходит код подтверждения
Проверьте SMTP настройки в api.php

Попробуйте тестовый email test@lexchat.rf.gd – код будет 131313

Проверьте спам

Сессия истекает через 2 часа
Увеличьте session.gc_maxlifetime в .htaccess или в php.ini (если доступно)

Или включите keep_alive как описано в документации по клиентской части
